## Part III Requirements

### Building the Power BI Dashboard

Objective

Create an interactive dashboard to present mission-critical business insights to upper management and company executives.

- **Data Sources**: Findings/results from Part II: EDA

<br>

**STEPS**

- Create the plan for the dashboard based on the summary page findings.
- “Connect the dots” and decide on the purpose and story the dashboard will tell.
- Connect SQL database to Power BI.
- Create calculated columns and measures using DAX accordingly in Power BI as needed to create dashboard visuals.
- Create all of the individual views within the dashboard

<br>

🧰**SKILLS DEMONSTRATED**🧰

- Data Modeling
- Dashboard Creation
- Power BI
- DAX syntax
- Story Telling

* * *

<br>

## 🏗️ Dashboards (dashboard planning)

The dashboards for this project are the following:

1. Executive Summary Dashboard: Track general business KPIs: big picture
2. Sales Territory Dashboard: Compare Regional Performance (geospatial analysis: map)
3. Product Summary Dashboard: Analyze Product Level Trends (product detail)
4. Customer Summary Dashboard: Identify High-Value Customers (customer detail)
